## The Question Only K9 Can Answer

> "After months of AI-assisted iteration, does the repository contain only files that are actively referenced â€” or has the agent been leaving residue behind?"

A filesystem diff shows which files exist. It cannot show whether those files *should* exist â€” whether the agent that created them was under a contract to clean them up.

K9Audit records both: what the agent did, and what it was supposed to do. The gap is the violation.

---

## Background

The K9Audit repository was developed iteratively over several months with Claude Code as the primary agent. Each iteration cycle involved:

- Generating new scripts to replace previous versions
- Producing intermediate debug outputs and trace files
- Installing Python packages that left build artifacts
- Opening files in editors that left temporary lock files

No explicit cleanup contract was enforced. The agent was never required to verify that residue from previous iterations had been removed before completing a task.

This audit applies CIEU instrumentation retroactively to detect what accumulated.

## Intent Contract (Y\*_t)

The following five rules were derived from established software engineering definitions (Martin Fowler, *Refactoring*; IEEE Std 1490) and encoded as the audit's intent contract:

| Rule | ID | Definition |
|---|---|---|
| 1 | `SUPERSEDED` | If a `*_fixed.py`, `*_v2.py`, or `*_old.py` variant exists, the original it supersedes must have been removed |
| 2 | `ORPHANED_TXT` | `.txt` files not referenced in any `.md` document are iteration residue |
| 3 | `ARTIFACT` | Office temp files (`~$*`), `.bak`, `.tmp` must not exist in the repository |
| 4 | `ORPHANED_JSONL` | `.jsonl` ledger files without a corresponding case `.md` document are residue |
| 5 | `UNREFERENCED_SCRIPT` | Root-level `.py` scripts not mentioned in README and not imported anywhere are residue â€” unless they follow a standalone naming convention (`test`, `replay`, `audit`) |

These rules were enforced via `@k9(deny_content=["VIOLATION"])` â€” every detected violation was recorded as a CIEU five-tuple in the audit ledger.

## Finding Analysis

### SUPERSEDED-01 â€” `k9_case002_replay_fixed.py`

During a Case 002 debugging session, an agent generated `k9_case002_replay_fixed.py` as a corrected version of `k9_case002_replay.py`. The agent's intent contract required it to either replace the original or delete the fixed copy after the correction was merged. Neither happened. Both files coexisted silently.

**Without K9**: Both files appear in `git ls-files`. No indication of which is canonical.  
**With K9**: The `SUPERSEDED` rule fires at the moment the fixed file is detected. The CIEU record shows the agent saw the naming pattern and failed to enforce cleanup.

### ORPHANED_TXT-01 â€” `k9audit_hook.egg-info/*.txt` (5 files)

Running `pip install -e .` generated a build artifact directory that was committed to the repository. These files are auto-generated by Python's setuptools on every install and should be excluded via `.gitignore`. They carry no information not already in `pyproject.toml`.

**Classification**: Build artifact residue. Not deleted â€” added to `.gitignore`.

### ORPHANED_TXT-02 â€” `challenge/examples/case_000_trace_step*.txt` (3 files) + `case_000_verify.txt`

During Case 000 development, intermediate `k9log trace` and `k9log verify-log` outputs were saved to disk to manually inspect ledger state. They were never cleaned up. They contain no information not reproducible by running `k9_live_test.py`.

**Classification**: Debug session residue. Deleted in Phase 2.

### ARTIFACT-01 â€” `docs/~$9_demo.html` and `server/~$shboard.html`

Microsoft Office (or a compatible editor) was used to open `k9_demo.html` and `dashboard.html`. The editor created lock files (`~$` prefix) that were accidentally staged and committed. These files contain no useful content.

**Classification**: Editor artifact. Deleted in Phase 2.

## What This Proves

This case demonstrates a capability no existing tool provides:

**Static analyzers** (pylint, SonarQube) detect code quality issues within files. They cannot detect whether a file *should exist at all* relative to an intent contract.

**Git history** shows what changed. It cannot show what the agent was *supposed to clean up* when it made a change.

**K9Audit** records the intent contract alongside the action. When an agent generates `k9_case002_replay_fixed.py`, the CIEU record captures: the agent knew this was a fix for an existing file; the contract required cleanup; the cleanup did not happen. The violation is causally traceable, not inferred.
