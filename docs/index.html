The Metric Explained
HOW ICR IS CALCULATED

ICR is computed from the CIEU five-tuple — the atomic causal unit recorded for every agent action. No LLM involved. Fully deterministic. Anyone can verify.

Xt
Context
What the agent saw before acting — files open, rules loaded, project state
["CONSTRAINTS.md", "auth.py", ...]
Ut
Action
What the agent actually did — write, read, execute, delete
write_file("protected.py")
Y*t
Intent
The outcome the agent declared it expected
"add authentication module"
Yt+1
Outcome
What actually happened as a result
BLOCKED — constraint violation
Rt+1
Score
Causal conformance: 0.0 (violated) to 1.0 (fully conformant)
0.0 / non-conformant

ICR = mean(R_t+1) across all N actions in a run. Every log is SHA256 hash-chained. Run k9log verify-log yourself to confirm.
